# 智能物流数据校验系统 - 简历项目描述

## 1. CheckByAi｜基于双Agent架构的智能物流数据校验系统

**项目角色：** 全栈开发（Agent架构核心负责人）  
**项目周期：** 2025.1 – 2025.2  
**技术栈：** Spring Boot 3.2、Vue 3、LangGraph、LangChain、Coze工作流、PaddleOCR、DeepSeek LLM、MyBatis-Plus、Redis、JWT、MySQL、Nginx

**项目描述：**
设计并实现双Agent架构的智能物流数据校验系统，支持Coze工作流（低代码平台）与LangGraph多Agent协作两种处理模式，用户可通过前端开关灵活切换。
Coze工作流模式通过API调用低代码平台实现快速部署，适合标准任务流程；LangGraph模式基于状态机工作流实现Excel解析、OCR识别、LLM校验的端到端自动化处理，适合复杂业务场景。
系统采用前后端分离架构，后端基于Spring Boot提供RESTful API，前端使用Vue 3实现文件上传、Agent模式切换、任务追踪、结果可视化等交互功能。支持批量数据处理（每批8条）、异步回调机制、任务状态实时追踪，处理准确率>95%，单次可处理数百条物流数据。

**核心工作与亮点：**
1. 设计并实现双Agent架构，集成Coze工作流（低代码平台）与LangGraph多Agent协作两种处理模式，前端提供可视化开关实现运行时动态切换，满足不同业务场景需求。
2. 实现LangGraph多Agent协作架构，构建Coordinator、Excel、OCR、Think四个专业化Agent，通过StateGraph实现状态机工作流与智能路由决策，支持条件路由与循环执行。
3. 实现CoordinatorAgent协调机制，根据执行状态（excel_rows、ocr_rows、report_text）动态决策下一步执行节点，确保工作流按需执行，提升处理效率。
4. 优化OCR批量处理流程，采用独立UUID目录策略（row{idx}_{side}_{uuid}）解决多行并发处理时的文件冲突问题，确保并发安全与资源隔离。
5. 设计专业化LLM Prompt工程，实现4层校验规则（完整性、逻辑性、一致性、格式规范），基于DeepSeek LLM进行重量一致性智能校验，生成结构化分析报告。
6. 解决OCR结果序列化问题，实现`_json_default`函数递归处理numpy类型（ndarray→list，标量→Python原生类型），确保JSON序列化安全与跨平台兼容。
7. 优化批次拆分算法，统一以excel_base.size()为准自动对齐excel_pull/excel_push数组长度，修复大数据量分批处理时的数据丢失问题，保证数据完整性。
8. 基于JWT + ThreadLocal实现无状态身份认证，支持BCrypt密码加密与MD5到BCrypt的平滑迁移，引入Redis缓存热点数据并设计缓存失效策略，提升系统安全性与响应性能。

## 补充说明（面试时可展开）
1. **双Agent架构设计**：Coze工作流与LangGraph的选型对比，如何通过统一接口实现两种模式的切换，前端开关的实现机制
2. **LangGraph工作流设计**：详细说明如何用StateGraph构建状态机，Coordinator如何做路由决策，状态传递机制
3. **OCR优化细节**：独立目录策略、错误容错机制、资源清理时机，并发安全的实现原理
4. **LLM Prompt设计**：4层校验规则的具体实现，如何保证输出格式一致性，异常检测准确率优化
5. **系统扩展性**：如何支持更多Agent（如数据清洗Agent、报告生成Agent），双Agent架构的扩展方案
