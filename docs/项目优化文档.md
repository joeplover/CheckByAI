## 本次代码检查与修复摘要（已落地）

- **密码安全**：后端 `AuthService` 已从 **MD5** 切换为 **BCrypt**；并支持“老MD5密码登录成功后自动升级为BCrypt”，避免一次性改库导致用户无法登录。
- **删除任务Bug修复**：`/api/task/{taskId}` 里的 `taskId` 实际是业务字段 `task.task_id`，原实现按主键 `id` 删除导致**删不掉任务**；已修复为按 `task_id` 删除。
- **Redis缓存真正启用**：`CacheConfig` 原来缺少 `@Configuration/@EnableCaching`，导致缓存管理器不生效；已补齐并对任务列表/任务结果增加最小缓存与回调写入后的缓存失效。
- **Redis序列化修正**：`RedisTemplate` 增加 value 的 JSON 序列化，避免 `opsForValue` 默认JDK序列化带来的不可读/体积大/跨语言不友好问题。
- **配置去敏**：`application.yml` 里不再硬编码 DB/Redis/JWT密钥/第三方Token，改为环境变量注入，并提供 `application-example.yml`。
- **前端可配置化**：前端不再写死 `API_BASE_URL`，改用 `VITE_API_BASE_URL`（示例在 `check_ai_web/env.example`）。
- **最小单元测试补齐**：新增 `AuthServiceTest`，覆盖 BCrypt 注册与 MD5→BCrypt 自动升级逻辑；集成测试类默认 `@Disabled`，避免无外部MySQL/Redis时CI失败。

## 后端关键环境变量（建议用部署环境注入）

- **REDIS_HOST / REDIS_PORT / REDIS_PASSWORD / REDIS_DATABASE**
- **JWT_SECRET**（生产务必强随机且至少32字节）
- **CHECKAI_WORKFLOW_API_URL / CHECKAI_WORKFLOW_BOT_ID / CHECKAI_WORKFLOW_AUTHORIZATION**
- **LANGCHAIN_AGENT_API**

## 后续优化建议（下一阶段）

- **统一返回结构**：目前后端同时存在 `Result{code,...}` 与 `{success:...}` 两种风格，建议统一，减少前端判断分支与兼容成本。
- **日志与异常治理**：回调处理里仍有 `System.out.println`，建议改为 `Slf4j` + 结构化日志；同时补充全局异常处理器统一输出错误码。
- **缓存策略精细化**：对热点接口（任务列表/结果）建议加“TTL抖动、防击穿/穿透”策略，详见 `docs/Redis热点缓存优化方案.md`。
- **消息化解耦**：上传→处理→回调链路建议引入RabbitMQ异步化与削峰，详见 `docs/RabbitMQ引入指南.md`。
